wouldn't it be nice if you could press one button and immediately download everyone's changes, and every version of them?
well THAT'S why i'm here

git is a fancy technology that lets you save and upload changes to files, instead of the whole file itself. once a file is tracked by git, saving it doesn't make a record of the entire thing, just of what is different between the last time you saved it and this time. because of this, it's incredibly lightweight and you save a lot of space both locally and when uploading.
multiple people can work on the same project, and since they're only uploading their "changes" instead of an entire file, as long as those changes don't contradict eachother, they can be consolidated into a functioning project that you can keep working on automaticallly.
in general this is called source control, and git is just one rendition of the idea, but it's pretty damn widely used, and it's the only one i've worked with. if you wanna use mercurial that's up to you

step five:
learn the words.
"repository" (repo) - a directory containing git data alongside a project
"commit" - make a backup of the changes between the last commit and the present. any commit can be accessed any time
"stage" - to prepare a file to be committed
"push" - upload commits to repository host so that other users may pull them
"pull" - download latest commits from the repo host
"branch" - a thread of updates stemming from some origin. multiple of these allow multiple variants of the project to be updated
"clone" - to download a repo with its git data

step six:
GET YOUR REPO AND START GOING
in whichever client you're using, find a big ol button that says CLONE
hit it, drop that link from before in as the source, and pick somewhere on your computer as the destination.
once you hit clone, that place will be a git repository and will automatically pick up any file changes that are made in there.
in your client, it'll also open a big area with a timeline of commits and changed files on the right (gitkraken) or in the file status menu (sourcetree).
you're basically good to go by this point. to start committing just change or add some files, and they'll show up in the unstaged area. hit stage, enter a message in the commit box, and you've made your commit backup save thing.
hit "push" to make the changes visible to other users of the repository, and "pull" to acquire the changes of others.
if there are things you haven't pulled yet, make sure you commit any changes before pulling, or it'll make a "stash", which is basically "hey i hid all of your changes since the last formal commit and this one so you could get fully synced with what youre pulling"
if you wanna get your changes out of the stash, you gotta "pop" (or apply) it. right click it on the timeline or whatever and press that.
