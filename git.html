HI GUYS

- HOW TO USE GIT EASY -

FOR YOUR AVERAGE DUDE

step one: https://git-scm.com
install it.
there will be a lot of options on the installer, if any of them matter to you do whatever, otherwise just keep clicking forward until you're done.
got it? great.

step two: https://github.com
make an account. secure password. whatever.
there's a cool chart that displays whenever you do commits (uploads). you can see mine at https://github.com/XanderAeon
try to make yours as cool.

step three:
get a repository.
if you're making a new repo hit the arrow in the top right of github and make it.
from there, you will be presented with a screen that says CLONE WITH THIS and a big link.
if you're using a repository that already exists, hit the green CODE button and grab that link.
regardless, hold onto it. we will use it shortly.

step four:
get a client.

step four point one (optional):
well technically you don't need a client you can just do the following sequence of events instead after having installed git and navigated to the path of the repository
git clone [link] (copies repo into folder)
git pull (acquire changes from github)
git add -A (prepares all changed files for a commit)
git commit -m [commit message] (saves the file changes)
git push (pushes saves to github)
but if you're here its probably because you know that sucks

REAL step four:
you have two options.
sourcetree (https://www.sourcetreeapp.com)
and gitkraken (https://www.gitkraken.com)
and don't let anyone tell you otherwise. especially not github.
sourcetree: totally free, open source, runs well, bad at merging, bad at cloning
gitkraken: free unless the project is hosted in a private repo, chromium (bad), really good at merging, looks pretty, has trello or some shit built in idk
i like gitkraken but my free student edition ran out so i'm on sourcetree right now. still pretty good.
the only thing you need to worry about while installing either is linking your github account so it has the correct credentials and knows where to push
also sourcetree tries to make you use bitbucket or mercurial or something stupid, just skip past all of that

step five:
learn the words.
"repository" (repo) - a directory containing git data alongside a project
"commit" - make a backup of the changes between the last commit and the present. any commit can be accessed any time
"stage" - to prepare a file to be committed
"push" - upload commits to repository host so that other users may pull them
"pull" - download latest commits from the repo host
"branch" - a thread of updates stemming from some origin. multiple of these allow multiple variants of the project to be updated
"clone" - to download a repo with its git data
